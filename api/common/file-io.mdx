## Overview

The `binario` namespace provides template functions for binary data I/O operations, primarily used for reading and writing PPM and CPPM image formats.

## Functions

### read_binary

Reads binary data of a specified type from an input stream.

```cpp
template<typename T>
T read_binary(std::istream & stream)
```

<ParamField path="stream" type="std::istream&" required>
  Input stream to read from
</ParamField>

<ResponseField name="return" type="T">
  The value read from the stream
</ResponseField>

**Throws**: `std::runtime_error` if reading fails

#### Example

<CodeGroup>
```cpp Reading 8-bit values
#include "common/binario.hpp"
#include <fstream>

std::ifstream file("image.ppm", std::ios::binary);
unsigned char red = binario::read_binary<unsigned char>(file);
unsigned char green = binario::read_binary<unsigned char>(file);
unsigned char blue = binario::read_binary<unsigned char>(file);
```

```cpp Reading 16-bit values
#include "common/binario.hpp"
#include <fstream>

std::ifstream file("image.cppm", std::ios::binary);
unsigned short red = binario::read_binary<unsigned short>(file);
unsigned short green = binario::read_binary<unsigned short>(file);
unsigned short blue = binario::read_binary<unsigned short>(file);
```
</CodeGroup>

### write_birary

Writes binary data of a specified type to an output stream.

```cpp
template<typename T>
void write_birary(std::ostream& stream, const T& value)
```

<ParamField path="stream" type="std::ostream&" required>
  Output stream to write to
</ParamField>

<ParamField path="value" type="const T&" required>
  Value to write to the stream
</ParamField>

#### Example

<CodeGroup>
```cpp Writing pixel data
#include "common/binario.hpp"
#include <fstream>

std::ofstream file("output.ppm", std::ios::binary);
unsigned char red = 255;
unsigned char green = 128;
unsigned char blue = 64;

binario::write_birary(file, red);
binario::write_birary(file, green);
binario::write_birary(file, blue);
```


</CodeGroup>

</ParamField>

  Photo object to store the loaded image data (SOA format)
</ParamField>

<ResponseField name="return" type="bool">
  Returns `true` if the image was loaded successfully, `false` otherwise
</ResponseField>

**Location**: `imgsoa/imagesoa.cpp:268`

#### Example

```cpp

imgsoa::Photo photo;
if (imgsoa::loadPhoto("input.ppm", photo)) {
```

### readHeader

Reads the PPM file header and validates the format.

```cpp
bool readHeader(std::ifstream& file, Photo& photo)
```

<ParamField path="file" type="std::ifstream&" required>
  Input file stream positioned at the start of the header
</ParamField>

<ParamField path="photo" type="Photo&" required>
  Photo object to store header information (magic number, width, height, max color value)
</ParamField>

<ResponseField name="return" type="bool">
  Returns `true` if the header was read successfully and format is P6, `false` otherwise
</ResponseField>

**Location**: `imgsoa/imagesoa.cpp:293`

#### Format Details

The function reads:
- Magic number (must be "P6" for binary PPM)
- Image width
- Image height
- Maximum color value (determines 8-bit or 16-bit encoding)

### readPixels (SOA)

Reads pixel data from a PPM file into Structure of Arrays format.

```cpp
template<typename T>
void readPixels(std::ifstream& file, size_t pixelCount, Photo &photo)
```

<ParamField path="file" type="std::ifstream&" required>
  Input file stream positioned at pixel data
</ParamField>

<ParamField path="pixelCount" type="size_t" required>
  Total number of pixels to read (width × height)
</ParamField>

<ParamField path="photo" type="Photo&" required>
  Photo object with pre-allocated red, green, and blue vectors
</ParamField>

**Template Parameters**:
- `T`: Data type for pixel values (`unsigned char` for 8-bit, `unsigned short` for 16-bit)

**Location**: `imgsoa/imagesoa.cpp:305`

#### Example

```cpp
size_t pixelCount = photo.width * photo.height;
photo.red.resize(pixelCount);
photo.green.resize(pixelCount);
photo.blue.resize(pixelCount);

if (photo.maxColorValue <= 255) {
    readPixels<unsigned char>(file, pixelCount, photo);
} else {
    readPixels<unsigned short>(file, pixelCount, photo);
}
```

## PPM Format Support

### Format Types

<ResponseField name="P6" type="Binary PPM">
  Binary format with RGB pixel data
</ResponseField>

### Color Depth

- **8-bit**: Max color value ≤ 255 (uses `unsigned char`)
- **16-bit**: Max color value > 255 (uses `unsigned short`)

### File Structure

1. Magic number ("P6")
2. Width and height (whitespace-separated integers)
3. Maximum color value
4. Binary pixel data (R, G, B sequence)

#### Example PPM Header

```text
P6
1920 1080
255
[binary pixel data follows]
```
