    ```bash
    ls build-release/imtool-aos/imtool-aos
    ls build-release/imtool-soa/imtool-soa
    ```
  </Step>
</Steps>

## Command Syntax

ImgTool follows this pattern for all operations:

```bash
./imtool-<variant> <input_file> <output_file> <operation> [parameters]
```

**Variants:**
- `aos` - Array of Structs implementation
- `soa` - Struct of Arrays implementation

<Note>
  Both variants produce identical results but have different performance characteristics. The SOA variant is typically faster for operations that process color channels independently.
</Note>

## Basic Operations

### Compress Images

Reduce file size using color palette optimization:

<CodeGroup>
```bash AOS Variant
./build-release/imtool-aos/imtool-aos \
  input/lake.ppm \
  output/lake-compressed.cppm \
  compress
```

```bash SOA Variant
./build-release/imtool-soa/imtool-soa \
  input/lake.ppm \
  output/lake-compressed.cppm \
  compress
```
</CodeGroup>

<Info>
  Compressed images use the `.cppm` extension (custom PPM format). The compress operation analyzes color frequency and optimizes the color palette.
</Info>

### Resize Images

Scale images to specific dimensions (width × height):

<CodeGroup>
```bash Resize to 800×600
./build-release/imtool-soa/imtool-soa \
  input/lake.ppm \
  output/lake-resized.ppm \
  resize 800 600
```

```bash Upscale to 8000×8000
./build-release/imtool-aos/imtool-aos \
  input/deer-small.ppm \
  output/deer-large.ppm \
  resize 8000 8000
```
</CodeGroup>

**Syntax:**
```bash
resize <width> <height>
```

Both upscaling and downscaling are supported with interpolation.

### Cut Frequency

Filter out low-frequency colors to create a posterized effect:

```bash
./build-release/imtool-aos/imtool-aos \
  input/image.ppm \
  output/filtered.ppm \
  cutfreq 50000
```

**Syntax:**
```bash
cutfreq <threshold>
```

<Note>
  The threshold parameter specifies the minimum number of times a color must appear to be retained. Colors appearing fewer times are removed.
</Note>

### Max Level Normalization

Normalize color values to a specified maximum level:

```bash
./build-release/imtool-soa/imtool-soa \
  input/image.ppm \
  output/normalized.ppm \
  maxlevel 65535
```

**Syntax:**
```bash
maxlevel <max_value>
```

Common values:
- `255` - 8-bit color depth
- `65535` - 16-bit color depth

<Info>
  This operation is useful for extending color depth and improving image contrast by stretching the color range.
</Info>

### Image Info

Display detailed information about an image:

```bash
./build-release/imtool-aos/imtool-aos \
  input/image.ppm \
  dummy.ppm \
  info
```

<Warning>
  The `info` command still requires an output filename (can be a dummy name) due to the argument parser design, but no output file is created.
</Warning>

**Output includes:**
- Image dimensions (width × height)
- Maximum color value
- Number of unique colors
- Color frequency statistics

## Performance Profiling

ImgTool can be profiled using `perf` to measure performance and energy consumption:

<CodeGroup>
```bash Compress Performance
perf stat -a -e power/energy-pkg/ \
  ./build-release/imtool-aos/imtool-aos \
  ./input/lake-large.ppm \
  ./output/compress-lake-large-aos.cppm \
  compress
```

```bash Resize Performance
perf stat -a -e power/energy-pkg/ \
  ./build-release/imtool-soa/imtool-soa \
  ./input/deer-small.ppm \
  ./output/resize-deer-small-soa.ppm \
  resize 8000 8000
```

```bash CutFreq Performance
perf stat -a -e power/energy-pkg/ \
  ./build-release/imtool-aos/imtool-aos \
  ./input/lake-large.ppm \
  ./output/cutfreq-lake-large-aos.ppm \
  cutfreq 100000
```
</CodeGroup>

<Note>
  The `-a -e power/energy-pkg/` flags enable system-wide energy package monitoring. This requires appropriate permissions and hardware support.
</Note>

**Metrics captured:**
- Execution time
- Energy consumption
- Cache behavior
- Memory bandwidth utilization

## Comparing AOS vs SOA

Benchmark both implementations to see performance differences:

```bash
# AOS variant
time ./build-release/imtool-aos/imtool-aos \
  input/large-image.ppm output/aos-out.ppm compress

# SOA variant  
time ./build-release/imtool-soa/imtool-soa \
  input/large-image.ppm output/soa-out.ppm compress
```

<Info>
  The SOA (Struct of Arrays) variant often performs better for operations that process color channels independently due to improved cache locality and vectorization potential.
</Info>

## Supported File Formats

**Input formats:**
- PPM P3 (ASCII)
- PPM P6 (binary)

**Output formats:**
- PPM (standard Portable Pixmap)
- CPPM (compressed PPM format for `compress` operation)

<Warning>
  ImgTool only supports PPM format images. Convert your images to PPM before processing:
  
  ```bash
  convert input.jpg output.ppm  # Using ImageMagick
  ```
</Warning>

## Example Workflow

Here's a complete workflow processing an image:

<Steps>
  <Step title="Get Image Information">
    ```bash
    ./build-release/imtool-aos/imtool-aos \
      input/photo.ppm output/dummy.ppm info
    ```
  </Step>

  <Step title="Resize for Web">
    ```bash
    ./build-release/imtool-soa/imtool-soa \
      input/photo.ppm output/photo-800.ppm \
      resize 800 600
    ```
  </Step>

  <Step title="Apply Artistic Effect">
    ```bash
    ./build-release/imtool-aos/imtool-aos \
      output/photo-800.ppm output/photo-artistic.ppm \
      cutfreq 10000
    ```
  </Step>

  <Step title="Compress for Storage">
    ```bash
    ./build-release/imtool-soa/imtool-soa \
      output/photo-artistic.ppm output/photo-final.cppm \
      compress
    ```
  </Step>
</Steps>

## Troubleshooting

### Wrong Number of Arguments

If you see:
```
Error : Wrong number of arguments; Needed minimum 3 parameters
```

Ensure you're providing all required arguments:
```bash
./imtool-aos <input> <output> <operation> [params]
```

### Invalid Operation

If you see:
```
Wrong operation introduced; <operation> is not a valid operation
```

Valid operations are:
- `compress`
- `resize`
- `cutfreq`
- `maxlevel`
- `info`

### File Not Found

Ensure your input file exists and paths are correct:
```bash
ls -la input/lake.ppm
```
