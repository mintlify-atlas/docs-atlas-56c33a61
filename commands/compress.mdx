The `compress` operation reduces image file size by building an optimized color palette and storing pixels as palette indices rather than full RGB values.

## Syntax

```bash
imtool-<variant> <input.ppm> <output.cppm> compress
```

<ParamField path="input.ppm" type="path" required>
  Input PPM file in P6 format
</ParamField>

<ParamField path="output.cppm" type="path" required>
  Output compressed file (CPPM format)
</ParamField>

## How It Works

### Algorithm Overview

1. **Read Input**: Load PPM file and parse header (magic number, dimensions, max color value)
2. **Build Color Table**: Scan all pixels and create a map of unique colors
3. **Index Assignment**: Assign sequential indices to each unique color (0 to n_colors-1)
4. **Write CPPM**: Output custom format with color table followed by pixel indices

      std::map<Pixel, uint32_t> color_table;
      
      // Find unique colors
      for (const auto& color : input) {
        if (color_table.find(color) == color_table.end()) {
          color_table[color] = 0;
          picture.colors++;
        }
      }
      
      // Assign indices
      uint32_t indice = 0;
      for (auto [color, index] : color_table) {
        color_table[color] = indice++;
      }
      
      return color_table;
    }

## CPPM Output Format

The compressed CPPM format uses the magic number `C6`:

```
C6 <width> <height> <max_color> <n_colors>
<color_table>
<pixel_indices>
```

### Header Structure

- **Magic Number**: `C6` (indicates compressed PPM)
- **Width**: Image width in pixels
- **Height**: Image height in pixels  
- **Max Color**: Maximum color value (255 or 65535)
- **N Colors**: Number of unique colors in palette

### Color Table

Each color is stored based on bit depth:

- **8-bit** (max_color ≤ 255): 3 bytes per color (R, G, B as `uint8_t`)
- **16-bit** (max_color > 255): 6 bytes per color (R, G, B as `uint16_t`)

### Pixel Indices

Each pixel stores an index into the color table:

- **n_colors ≤ 255**: 1 byte per pixel (`uint8_t`)
- **256 ≤ n_colors ≤ 65535**: 2 bytes per pixel (`uint16_t`)
- **n_colors > 65535**: 4 bytes per pixel (`uint32_t`)

## Compression Ratio

The compression ratio depends on the number of unique colors:

**Original PPM size**: `width × height × 3 bytes` (for 8-bit)

**CPPM size**: `header + (n_colors × 3) + (width × height × index_size)`

Best compression occurs when:
- Image has few unique colors (e.g., graphics, logos)
- Large images with repeated color patterns

<Note>
  For photographic images with many unique colors, CPPM may be larger than the original PPM. This format is optimized for images with limited color palettes.
</Note>

## Performance Characteristics

- **Time Complexity**: O(n) for scanning pixels + O(k log k) for building sorted color map (where k = unique colors)
- **Memory**: Requires loading entire image into memory
- **I/O**: Two passes - one read (input PPM), one write (output CPPM)

## Error Conditions

<Warning>
  The compress operation will fail if:
  
  - Input file is not valid P6 PPM format
  - Cannot open input file for reading
  - Cannot create output file for writing
  - Insufficient memory to load image
</Warning>
