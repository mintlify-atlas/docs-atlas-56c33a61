<Warning>
**Implementation Status**: The `info` command is currently **not connected** in the main executables (`imtool-aos/main.cpp` and `imtool-soa/main.cpp`). While the function exists in `common/info.cpp` and is listed as a valid operation in the argument parser, it is not invoked when you run the command. This is a known limitation in the current source code.

To use this functionality, you would need to modify the main.cpp files to add an `else if` clause that calls `common::info(params)` when the operation is "info".
</Warning>

The `info` operation is designed to display metadata about a PPM image file without modifying it. This is useful for inspecting image properties before processing.

## Syntax

```bash
imtool-<variant> <input.ppm> <output_dummy> info
```

<ParamField path="input.ppm" type="path" required>
  Input PPM file in P6 format
</ParamField>

<ParamField path="output_dummy" type="path" required>
  Dummy output path (required but not used)
</ParamField>

<Note>
  The `info` command requires an output filename argument for consistency with other commands, but it does not create or modify any files.
</Note>

## Implementation

**Function**: `common::info()`

Location: `common/info.cpp:8-28`

```cpp
void info(const progargsCommon::parameters_files& params) {
  std::ifstream imagen(params.input_file, std::ios::binary);
  if (imagen.fail()) {
    std::cerr << "Error opening file " << params.input_file << '\n';
    return;
  }
  
  // Read PPM header
  std::string identification;
  int width = 0;
  int height = 0;
  int max_color = 0;
  
  imagen >> identification;
  if (identification != "P6") {
    std::cerr << "Invalid PPM format\n";
    return;
  }
  
  imagen >> width >> height >> max_color;
  
  // Display information
  std::cout << "Input: " << params.input_file << '\n'
            << "Output: " << params.output_file << '\n'
            << "Operation: info\n"
            << "Image size: " << width << "x" << height << '\n'
            << "Max level: " << max_color;
}
```

## Output Information

The info command displays:

1. **Input file path**: Full path to the image being inspected
2. **Output file path**: The dummy output path provided
3. **Operation**: Confirms "info" operation
4. **Image size**: Width and height in pixels (format: WIDTHxHEIGHT)
5. **Max level**: Maximum color value (255 for 8-bit, 65535 for 16-bit)

## Interpreting the Output

### Image Size

```
Image size: 3840x2160
```

- **Width**: 3840 pixels
- **Height**: 2160 pixels
- **Total pixels**: 3840 × 2160 = 8,294,400

### Max Level (Bit Depth)

```
Max level: 255
```

| Max Level | Bit Depth | Bytes/Channel | File Size Formula |
|-----------|-----------|---------------|-------------------|
| 255 | 8-bit | 1 | width × height × 3 |
| 65535 | 16-bit | 2 | width × height × 6 |

**File size calculation**:

```
8-bit (255):   3840 × 2160 × 3 = 24,883,200 bytes ≈ 24 MB
16-bit (65535): 3840 × 2160 × 6 = 49,766,400 bytes ≈ 48 MB
```

## Practical Use Cases

### 1. Pre-processing Inspection

Check image properties before applying operations:

```bash
# Check if image is already 16-bit
./imtool-aos photo.ppm dummy.ppm info
# Output: Max level: 255

# Extend to 16-bit if needed
./imtool-aos photo.ppm photo-16bit.ppm maxlevel 65535
```

### 2. Verify Image Dimensions

Confirm dimensions match expected values:

```bash
./imtool-soa downloaded.ppm dummy.ppm info
# Expected: 1920x1080
# Actual: Image size: 1920x1080  ✓
```

### 3. Batch Processing Validation

Verify all images in a directory:

```bash
#!/bin/bash
for img in ./input/*.ppm; do
  echo "Processing: $img"
  
  # Get info
  info_output=$(./imtool-aos "$img" dummy.ppm info)
  
  # Check if 8-bit
  if echo "$info_output" | grep -q "Max level: 255"; then
    echo "  Converting to 16-bit..."
    output="./output/$(basename "$img")"
    ./imtool-aos "$img" "$output" maxlevel 65535
  fi
done
```

### 4. Calculate Processing Time

Estimate processing time based on image size:

```bash
./imtool-aos large-image.ppm dummy.ppm info
# Image size: 15360x8640
# Estimate: Large image, resize will take ~30 seconds
```

## Performance Characteristics

- **Time Complexity**: O(1) - only reads header
- **Memory**: Minimal - no pixel data loaded
- **I/O**: Only header is read (first ~50 bytes)
- **Speed**: Near-instantaneous even for huge files

<Note>
  The info command is extremely fast because it only reads the PPM header, not the actual pixel data.
</Note>

## Error Handling

<Warning>
  The info operation will fail if:
  
  - Input file does not exist or cannot be opened
  - File is not in P6 PPM format
  - PPM header is malformed
</Warning>

**Error messages**:

```bash
# File not found
Error opening file ./input/missing.ppm

# Invalid format
Invalid PPM format

# Not a P6 file (e.g., P3 ASCII format)
Invalid PPM format
```

## Limitations

<Note>
  The info command **does not** display:
  
  - Number of unique colors (use external tools like `identify` from ImageMagick)
  - Color depth per channel (always assumes RGB)
  - Compression ratio (CPPM files not supported)
  - EXIF metadata (PPM format doesn't store EXIF)
  - Color profile information
</Note>

## Supported Formats

**Supported**: P6 (binary PPM)

```
P6
<width> <height>
<max_color>
<binary RGB data>
```

**Not supported**:
- P3 (ASCII PPM)
- CPPM (compressed format from ImgTool compress command)
- PGM (grayscale)
- PBM (bitmap)

## Comparison with External Tools

### ImageMagick `identify`

```bash
$ identify -verbose image.ppm
```

Provides much more information:
- Unique colors count
- Color histograms  
- Mean/standard deviation
- File size
- EXIF data (for formats that support it)

### ImgTool `info`

```bash
$ ./imtool-aos image.ppm dummy.ppm info
```

Provides basic information:
- Dimensions
- Max color value
- Fast and lightweight
- Consistent with ImgTool command structure
