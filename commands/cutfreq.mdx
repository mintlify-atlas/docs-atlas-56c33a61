---
title: 'cutfreq'
description: 'Remove least frequent colors to create posterization effects'
---

The `cutfreq` operation reduces the number of unique colors in an image by removing the least frequently occurring colors and replacing them with the nearest remaining colors. This creates a posterization effect.

## Syntax

```bash
imtool-<variant> <input.ppm> <output.ppm> cutfreq <threshold>
```

<ParamField path="input.ppm" type="path" required>
  Input PPM file in P6 format
</ParamField>

<ParamField path="output.ppm" type="path" required>
  Output posterized PPM file
</ParamField>

<ParamField path="threshold" type="integer" required>
  Number of least frequent colors to remove (must be ≥ 1)
  
  Typical values:
  - `1000` - Subtle reduction
  - `100000` - Noticeable posterization
  - `500000` - Strong posterization effect
</ParamField>

## How It Works

### Frequency Filtering Algorithm

1. **Count Color Frequency**: Build histogram of all colors in the image
2. **Identify Least Frequent**: Sort colors by frequency and select bottom N
3. **Build Spatial Grid**: Create 3D grid for efficient nearest-color lookup
4. **Replace Colors**: Map each removed color to its nearest remaining color
5. **Apply Mapping**: Update all pixels that used removed colors

          pixel = cacheIter->second;

          auto iter = replacementMap.find(pixel);
          if (iter != replacementMap.end()) {
            pixel = iter->second;
            colorCache[pixel] = iter->second;
          }
        }
      }
    }

    std::vector<Pixel> getLeastFrequentColors(
        std::unordered_map<Pixel, int, PixelHash> frequencyMap, int n) {
      
      std::vector<std::pair<Pixel, int>> colorFrequency(
        frequencyMap.begin(), frequencyMap.end());
      
      // Sort by frequency (ascending), then by B,G,R (descending)
      std::ranges::sort(colorFrequency, [](auto const& a, auto const& b) {
        if (a.second != b.second) return a.second < b.second;
        if (a.first.b != b.first.b) return a.first.b > b.first.b;
        if (a.first.g != b.first.g) return a.first.g > b.first.g;
        return a.first.r > b.first.r;
      });
      
      std::vector<Pixel> colorsToRemove;
      size_t numColorsToRemove = std::min((size_t)n, colorFrequency.size());
      colorsToRemove.reserve(numColorsToRemove);
      
      for (size_t i = 0; i < numColorsToRemove; ++i) {
        colorsToRemove.push_back(colorFrequency[i].first);
      }
      
      return colorsToRemove;
    }

    // Build 3D spatial grid (8×8×8 cells)
    std::unordered_map<int, std::vector<Pixel>> 
    buildGrid(const std::vector<Pixel>& image, int gridSize) {
      std::unordered_map<int, std::vector<Pixel>> grid;
      for (const auto& pixel : image) {
        int key = ((pixel.r / gridSize) * 10000) + 
                  ((pixel.g / gridSize) * 100) + 
                  (pixel.b / gridSize);
        grid[key].push_back(pixel);
      }
      return grid;
    }
    
    // Find closest color using grid (checks 27 neighboring cells)
    Pixel findClosestColorOptimized(const Pixel& color,
        const std::unordered_map<int, std::vector<Pixel>>& grid,
        int gridSize) {
      
      int key = ((color.r / gridSize) * 10000) + 
                ((color.g / gridSize) * 100) + 
                (color.b / gridSize);
      
      Pixel closestColor = {0, 0, 0};
      double minDistanceSquared = std::numeric_limits<double>::max();
      
      // Check 3×3×3 neighborhood in grid
      for (int dr = -1; dr <= 1; ++dr) {
        for (int dg = -1; dg <= 1; ++dg) {
          for (int db = -1; db <= 1; ++db) {
            int neighborKey = key + (dr * 10000) + (dg * 100) + db;
            
            if (grid.find(neighborKey) != grid.end()) {
              for (const auto& neighborColor : grid.at(neighborKey)) {
                // Euclidean distance squared
                double distSq = pow(color.r - neighborColor.r, 2) +
                                pow(color.g - neighborColor.g, 2) +
                                pow(color.b - neighborColor.b, 2);
                
                if (distSq < minDistanceSquared) {
                  minDistanceSquared = distSq;
                  closestColor = neighborColor;
                }
              }
            }
          }
        }
      }
      return closestColor;
    }

      for (size_t i = 0; i < image.width * image.height; ++i) {

        int bIndex = image.b[i] / gridSize;

## Grid Optimization

The spatial grid divides RGB color space into 8×8×8 cells:

```
Grid cell size: 255 / 8 = 32 color values per dimension

Color (128, 200, 50) maps to cell:
  r_index = 128 / 32 = 4
  g_index = 200 / 32 = 6
  b_index = 50 / 32 = 1
  
  key = (4 × 10000) + (6 × 100) + 1 = 40601
```

When finding nearest color, only 27 neighboring cells are checked (3×3×3 cube), significantly faster than checking all remaining colors.

## Color Distance Metric

Cutfreq uses **Euclidean distance** in RGB space to find the nearest color:

```cpp
distance = sqrt((r1-r2)² + (g1-g2)² + (b1-b2)²)
```

Actually computed as squared distance to avoid expensive `sqrt()`:

```cpp
distanceSquared = (r1-r2)² + (g1-g2)² + (b1-b2)²
```

<Note>
  RGB distance doesn't perfectly match human color perception. For better results, consider converting to LAB color space first (not implemented in ImgTool).
</Note>

## Sorting Behavior

When colors have the same frequency, they're sorted by RGB values (descending):

```cpp
if (a.second != b.second) return a.second < b.second;  // frequency
if (a.first.b != b.first.b) return a.first.b > b.first.b;  // blue
if (a.first.g != b.first.g) return a.first.g > b.first.g;  // green
return a.first.r > b.first.r;  // red
```

This ensures deterministic results when multiple colors have the same frequency.

## Performance Characteristics

- **Time Complexity**: 
  - Counting: O(n) where n = pixels
  - Sorting: O(k log k) where k = unique colors
  - Grid building: O(k)
  - Replacement: O(n) with grid optimization
  - Overall: O(n + k log k)

- **Space Complexity**: 
  - Frequency map: O(k)
  - Grid: O(k)
  - Replacement map: O(threshold)
  - Overall: O(k + threshold)

<Note>
  Grid optimization reduces nearest-color search from O(k) to approximately O(1) by limiting search to 27 cells.
</Note>

## Use Cases

### 1. Pre-compression

Reduce colors before compress operation for better compression ratios:

```bash
# First reduce colors
./imtool-aos input.ppm reduced.ppm cutfreq 50000

# Then compress
./imtool-aos reduced.ppm output.cppm compress
```

### 2. Artistic Effects

Create poster-style or pop art effects:

```bash
./imtool-soa photo.ppm poster-art.ppm cutfreq 200000
```

### 3. Noise Reduction

Remove color noise from scanned images:

```bash
./imtool-aos scanned.ppm clean.ppm cutfreq 500
```

  The cutfreq operation will fail if:

  - Input file is not valid P6 PPM format
  - Insufficient memory for frequency map
  - Cannot create output file

## Comparison with Other Tools

**ImgTool cutfreq** vs **ImageMagick -posterize**:

- cutfreq: Removes least frequent colors, preserves common colors exactly
- posterize: Reduces colors per channel uniformly

cutfreq is better for:
- Maintaining dominant colors
- Noise reduction
- Pre-compression optimization
